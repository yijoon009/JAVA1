메소드
	이름 뒤에 소괄호()
	키워드 뒤에 소괄호는 메소드가 아니다.

	함수
	f(x)=2x+1
	f(3)=7	

	f: 메소드 이름
	x: 매개변수
	2x+1: 리턴 값

메소드 선언

	리턴타입 메소드명(자료형 매개변수1,,,){
		실행할 문장;
		return 리턴값;
	}

	1.리턴타입: 리턴값의 타입을 결정하는 부분, 리턴값이 없으면 void
	2.메소드명: 동사형으로 작성한다.
	3.자료형: 외부에서 전달할 값이 없다면 생략 가능하다.
	외부에서 전달해 주어야 하는 값의 갯수만큼 선언해 주어야 한다. 사용시에는 매개변수의 타입과 갯수에 맞게 순서대로 값을 전달해 주어야 한다. (,로 구분)
	4.실행할 문장: 생략 가능하다.
	5.리턴: 사용하는 부분으로 돌려줄 값이 없다면 생략이 가능하다.
	리턴 뒤에 값이 있다면, 메소드를 사용하는 부분 통채로가 그 리턴값으로 치환된다.



* {} 중괄호는 이 메소드를 만든다/선언한다

메소드 선언 순서

	문제) 두 정수의 곱셈
	
	1)메소드 명을 생각한다.
		void mul(){
	
		}
	2)매개변수를 생각한다.
		void mul(int n1, int n2){
	
		}
	3)실행할 문장을 생각한다.
		void mul(int n1, int n2){
			int result=n1*n2;
			syso("두 정수의 곱셈");
		}
	4)리턴의 여부를 생각한다.
		void mul(int n1, int n2){
			int result=n1*n2;
			syso("두 정수의 곱셈");
			return result;
		}
	5)리턴타입을 결정한다.
		int mul(int n1, int n2){
			int result=n1*n2;
			syso("두 정수의 곱셈");
			return result;
		}

메소드 사용
	메소드명(매개값);
	-매개변수 타입과 개수에 맞게 넘겨준다.

메소드 주의사항
	클래스 영영 밖에서 선언불가, 사용불가
	메소드 영역 안에서 선언 불가
	메소드 영역 밖에서 사용 불가

※메소드를 선언할 때에는 반드시 메소드 밖에서 선언하고, 사용은 반드시 안에서 사용해야 한다.

리턴을 사용해야 할 때
	사용하는 부분에서 그 결과값으로 처리해야할 행위가 남았을 때
	메소드의 처리 결과를 사용하는 부분에 알려주어야 할 때

리턴을 사용하지 않아도 될때
	메소드에서 모든 기능이 구현될 때

메소드의 사용 목적
	1.반복되는 코드, 반복되는 기능을 쉽게 관리하기 위하여.
	재사용의 목적이 있기 때문에 특정성이 부여되면 안된다.

	2.소스코드 간결화
